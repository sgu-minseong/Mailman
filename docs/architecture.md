\# Architecture



\## 역할 분리



이 프로젝트는 n8n을 중심으로 행정 문의 처리 흐름을 자동화하고, Lovable로 직원 검토용 대시보드를 구성했습니다.



\## 구성 요소



\### n8n



전체 자동화 흐름의 중심입니다.



\- 메일 입력 처리

\- 첨부파일 여부 확인

\- Upstage API 호출

\- Solar LLM 호출

\- confidence 기반 분기

\- 직원 검토 대상으로 전달



\### Upstage Document Parse



첨부 서류를 분석해 텍스트 정보를 추출합니다.



예:

\- 신청서

\- 증명서

\- 등록금 관련 서류

\- 장학금 관련 서류



\### Upstage Information Extract



문서와 메일에서 행정 처리에 필요한 구조화된 정보를 추출합니다.



예:

\- 학생 정보

\- 문의 유형

\- 요청 내용

\- 관련 부서 후보



\### Solar LLM



추출된 정보를 바탕으로 담당 부서를 분류하고 답변 초안을 생성합니다.



예:

\- 장학 관련 문의

\- 등록금 관련 문의

\- 학적 관련 문의

\- 기숙사 관련 문의

\- 입학 관련 문의

\- 수동 검토 필요 문의



\### Lovable Dashboard



해커톤 필수 활용 조건에 맞춰 제작한 직원 검토용 UI입니다.



직원은 대시보드에서 다음 내용을 확인합니다.



\- 문의 요약

\- 추출된 학생 정보

\- AI가 분류한 담당 부서

\- 답변 초안

\- 수동 검토 필요 여부



\## 안전장치



AI 결과를 그대로 자동 처리하지 않고, 다음 경우에는 직원 검토로 넘기도록 설계했습니다.



\- 담당 부서 분류 confidence가 낮은 경우

\- 첨부파일 분석에 실패한 경우

\- 문의 유형이 불명확한 경우

\- 답변 초안 생성에 필요한 정보가 부족한 경우



\## Lovable의 역할



Lovable은 전체 서비스의 핵심 로직을 담당한 것이 아니라, 직원이 AI 분석 결과를 확인하는 대시보드 UI 제작에 사용했습니다.



전체 자동화 로직은 n8n 워크플로우에서 구성했습니다.
