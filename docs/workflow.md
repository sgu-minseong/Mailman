\# n8n Workflow



이 프로젝트의 핵심은 대학 행정 문의 메일을 AI로 분석하고, 담당 부서 분류와 답변 초안 생성을 자동화하는 n8n 워크플로우입니다.



\## 전체 처리 흐름



1\. 메일 입력 수신

&#x20;  - 행정실로 들어온 문의 메일을 입력으로 받습니다.

&#x20;  - 제목, 본문, 발신자, 첨부파일 여부를 확인합니다.



2\. 첨부파일 확인

&#x20;  - 첨부 서류가 있는 경우 문서 분석 단계로 넘깁니다.

&#x20;  - 첨부파일이 없는 경우 제목과 본문 중심으로 문의 유형을 판단합니다.



3\. 문서 분석

&#x20;  - Upstage Document Parse를 활용해 첨부 서류의 내용을 분석합니다.

&#x20;  - 학생이 제출한 서류나 증명서 내용을 텍스트로 변환해 이후 판단에 활용합니다.



4\. 정보 추출

&#x20;  - Upstage Information Extract를 활용해 학생 정보, 문의 유형, 핵심 요청 내용을 추출합니다.



5\. 담당 부서 분류

&#x20;  - Solar LLM이 메일 본문과 첨부파일 분석 결과를 바탕으로 담당 부서를 분류합니다.

&#x20;  - 예: 장학, 등록금, 학적, 기숙사, 입학, 수동 검토



6\. 답변 초안 생성

&#x20;  - 문의 유형에 맞는 답변 초안을 생성합니다.

&#x20;  - 직원은 초안을 그대로 보내는 것이 아니라 검토 후 수정할 수 있습니다.



7\. 신뢰도 기반 분기

&#x20;  - 담당 부서 분류 신뢰도가 낮거나 문서 분석에 실패한 경우 자동 처리하지 않습니다.

&#x20;  - 이 경우 직원 검토 대상으로 분류합니다.



8\. 직원 검토

&#x20;  - 직원은 대시보드에서 AI가 분류한 담당 부서, 추출 정보, 답변 초안을 확인합니다.

&#x20;  - 필요 시 수정 후 부서 라우팅 또는 답변 발송을 진행합니다.



\## 설계 포인트



\- AI가 모든 문의를 자동 처리하지 않도록 설계했습니다.

\- 문서 분석 실패, 낮은 confidence, 애매한 문의는 직원 검토로 넘겨 실제 행정 환경에서의 안전성을 고려했습니다.

\- 단순 분류가 아니라 문서 분석, 정보 추출, 담당 부서 판단, 답변 초안 생성까지 이어지는 행정 워크플로우 자동화를 목표로 했습니다.
